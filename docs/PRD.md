# 风格票根 PRD

## 1. 产品愿景

建立一本可以查询、理解、组合和沉淀视觉风格的桌面图鉴，让不熟悉 UI 的使用者也能把模糊的审美偏好转化为可感知的例图、可执行的设计规则和可复制给 AI 的自然语言提示词。

产品不是一份静态术语表，而是三种工具的结合：

- 视觉风格图鉴：看懂风格、技法、材质和组件。
- 提示词配方书：把审美判断翻译为可执行语言。
- Lab 风格实验室：组合多个元素并沉淀个人风格。

## 2. 目标用户与场景

核心用户是不具备系统 UI 训练、但需要借助 Codex、Claude Code、Lovable 等工具制作网页或应用的人。

典型场景：

1. 用户在电脑侧边打开小窗，搜索一个不熟悉的视觉名词。
2. 用户先看实时样张，再阅读定义、特征和常见组件。
3. 用户复制自然语言提示词，交给 AI 生成网页。
4. 用户把多个喜欢的元素加入 Lab，设定主次权重并生成融合配方。
5. 用户把满意的实验保存到“我的风格”，逐渐形成个人视觉语言。

## 3. 设计原则

- 图先于术语：先让用户一眼感知差异，再解释概念。
- 规则先于形容词：少用“高级、好看”，多给布局、材质、字体、色彩和禁忌。
- 小窗但不拥挤：默认 340 × 720 px，可纵向滚动；实际阅读文字不低于 10px，不为追求单屏塞满而牺牲可读性。
- 拟物服务信息：票根和纸张隐喻提供情绪，不牺牲可读性。
- 允许从模糊开始：个人风格可以暂不命名，逐步补充。
- 组合不是标签相加：Lab 必须表达主次，并提示潜在冲突。

## 4. 视觉方向

综合四张参考图形成以下方向：

- 借用风格展示板的组织方法：标题、短定义、组件样张、色板和关键词。
- 借用电影票根的收藏隐喻：顶部海报区、检票虚线、编号、齿孔和票面信息。
- 借用真实热敏小票的材质：暖白纸、轻微噪点、窄体文字、紧凑分隔和黑色印刷。
- 保留少量彩色墨印作为当前风格的识别色，不把全局界面染成对应风格。
- 避免做成现代 SaaS 卡片墙，也避免为了拟真加入影响阅读的污渍和扭曲。

## 5. 信息架构

### 图鉴

- 首页 / 今日票根
- 全部词条
- 按类型：风格体系、材质与技法、布局、组件、时代与平台
- 模糊搜索：中文别名、英文近似拼写、多关键词组合
- 从网页收录待解析风格，并保留来源；用户只需填写必填网址和选填备注
- 分类条可折叠，在图鉴页通过弹窗新增分类
- 在词条详情中调整所属分类
- 长按票根直接拖动展示顺序
- 收藏（后续）

### Lab

- 已选元素
- 权重调整
- 应用场景
- 情绪词
- 必须保留 / 禁止出现
- 中英文配方
- 冲突提示
- 保存为个人风格

### 我的风格

- 已保存实验
- 从例图收录个人风格
- 从文字规则、Prompt、tokens 或 Skill 内容录入个人风格
- 将来源解析为可编辑的设计规则和中英文 prompt
- 版本与备注（后续）
- 导出 JSON / Markdown / Skill（后续）

## 6. MVP 功能

| 功能 | 优先级 | 验收标准 |
|---|---:|---|
| 票根式窄窗 | P0 | 320px 宽主界面在 1366×768 桌面可侧边放置 |
| 固定底部导航 | P0 | 图鉴、Lab、我的三页切换时位置完全不变 |
| 窗口尺寸预设 | P0 | 支持紧凑、标准与长票三档，并记住选择 |
| 始终置顶 | 桌面版 | 可由用户切换是否钉在其他普通窗口上方 |
| AI 网关设置 | P0 | 设置页管理提供商、模型路由、地址与单次预算；不保存明文 Key |
| 词条浏览 | P0 | 可从目录进入详情并返回 |
| 搜索 | P0 | 中文名、英文名、标签均可匹配 |
| 模糊搜索 | P0 | 支持常用别名、英文轻微拼写错误和多个关键词 |
| 网页风格收录 | P0 | 网址必填、备注选填，其余字段由 AI 生成草稿 |
| 分类管理 | P0 | 分类条可展开；用户可弹窗新增，并在详情中重新分配 |
| 词条排序 | P0 | 长按票根后直接上下拖动并保存顺序 |
| 视觉样张 | P0 | 每个词条至少有一个实时 CSS 示意 |
| 复制提示词 | P0 | 一键复制当前词条英文提示词 |
| 加入 Lab | P0 | 任意词条可添加或移除，状态清晰 |
| 自由份量调整 | P0 | 每根滑块可独立设置 0–100，拖动时不改变其他滑块位置 |
| 后台权重归一化 | P0 | 系统根据全部自由份量换算实际占比，输出始终合计 100% |
| Lab 上限 | P0 | 最多同时加入 4 个风格，并提示 2–3 个最稳定 |
| 配方生成 | P0 | 根据选项和权重生成中英文提示词 |
| 保存个人风格 | P0 | 使用本地存储保留实验 |
| 独立收录个人风格 | P0 | 只保留例图提取与文字录入两个入口，文字可直接保存或选择 AI 整理 |
| 个人风格解析草稿 | P0 | 明确来源状态，并在未接 AI 时不伪造视觉结论 |
| 个人风格封面 | P0 | 例图生成本地压缩封面；Lab 配方使用参与风格样张 |
| 个人风格详情 | P0 | 可回看来源、规则、配方与中英文 Prompt |
| 编辑与删除 | P0 | 详情右上角可编辑内容和封面；删除前二次确认 |
| 冲突分析 | P1 | MVP 先使用规则检测明显冲突 |
| AI 个人风格解析 | 后续 | 视觉模型提取，文本模型归纳，用户确认后保存 |
| 图片生成 | 后续 | 可根据实验生成视觉样张 |
| AI 解释与命名 | 后续 | 接入模型生成融合说明和名称 |
| Windows 原生壳 | 后续 | 支持置顶、贴边、开机启动和系统托盘 |

## 7. 核心流程

### 查阅词条

打开应用 → 浏览或搜索 → 点击票根 → 查看样张与解释 → 复制提示词或加入 Lab。

### 创建个人风格

选择多个词条 → 打开 Lab → 调整权重 → 指定场景和情绪 → 查看冲突 → 生成配方 → 保存为个人风格。

## 8. 非目标

- MVP 不追求覆盖全部设计史。
- MVP 不作为严格的学术辞典。
- MVP 不接入账号、云同步或协作系统。
- MVP 不直接生成完整生产代码。
- MVP 不模仿某一张票据或某一品牌的受保护视觉资产。

## 9. 内容质量标准

每个词条必须同时包含：中文名、英文名、类型、简明定义、视觉特征、常见元素、适用场景、避免事项、中英文提示词、关联词条和可渲染样张类型。

提示词必须包含可执行约束，不能只堆叠抽象风格词。
