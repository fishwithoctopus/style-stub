# Contributing to Style Stub

感谢你帮助这本视觉风格图鉴变得更准确、更好用。

## 可以贡献什么

- 新的视觉风格、技法、材质、布局或组件词条
- 对定义、历史范围和关联关系的纠正
- 更可执行、可迁移的中英文 UI Prompt
- 原创实时 CSS 样张
- 可访问性、键盘操作和窄窗布局改进
- Lab 冲突规则、导入导出与桌面体验

## 词条要求

1. 定义要说明视觉机制，不只使用“高级、现代、好看”等形容词。
2. Prompt 要覆盖可执行的排版、颜色、材质、层级、组件与交互约束。
3. `avoid` 要指出最常见的误用。
4. 优先使用原创 CSS 样张；必须使用图片时，请确认许可并记录来源。
5. 品牌或平台案例必须注明快照日期与来源，不能把社区称呼写成官方定义。
6. 风格 Prompt 只能提取可迁移的视觉规则，不应照搬来源产品的文案、导航、图标含义或信息架构。

## 本地检查

```powershell
npm install
npm run check
npm test
npm run desktop
```

请至少检查图鉴列表、词条详情、Lab、个人馆藏和设置页；视觉或交互变化请在 Pull Request 中附上截图。

## 提交流程

1. Fork 仓库并创建范围清晰的分支。
2. 完成修改和本地检查。
3. 提交 Pull Request，说明变化、依据、视觉影响和验证结果。

## 个人数据与素材

不要提交浏览器本地存储、个人风格、私人提示词、API Key、访问令牌、电脑路径或未获许可的参考图。提交前请再次检查截图和日志。

## 讨论方式

设计术语可以有不同解释。请尽量给出来源，区分历史事实、社区惯用称呼与个人审美判断，并对新贡献者保持友善。
